What is git?

Created by Linus Torvalds for the Linux Kernel Source

Distributed Version Control Software

Snapshots vs Differences of file systems

Why is Version Control Software (VCS) Important?

  1. Lifesaver if your code stops working
  2. Good version control history is a story of your software
  3. Github is "street cred" for programmers
it shows where it began, major milestones, and your thought process as well as the collaboration with others

The Basics

Commit

“A Snapshot in time of your directory”

Unique hashcode

One or more parent commits

Additional MetaData (Author, Date, etc) 

git commit -m "My First Commit"

Commit Tree 

git log --graph --all # shows a pretty graph with all branches

Staging (Index)

“Where files go before they become a commit”

Move files to be staged (green) and un-staged (red)

Un-staged includes untracked files (files git doesn’t know about) 

git add newFile # add newFile to index (and track if not tracked)git add -u # Add all currently tracked filesgit add . # Add all files in directory (recursive)git reset HEAD newFile # Remove newFile from the index

Branches

Reference a commit by a name

Branches move as you commit

Work separately on something until its ready

  • bug fixes, features, etc. Read Gitflow!

Tags are just branches that don't move! 

git branch -a # Display all branchesgit checkout -b new-branch # Creates and checks out new-branchgit checkout bug-fix # Checkout bug-fix branch

Remotes

Up until now everything was local!

But we want to share our beautiful code with the world!

Can have unlimited remotes, usually just ‘origin' 

git remote add origin https://github.com/user/repo.gitgit fetch # Fetches updates from remotes but doesn't applygit push origin -u featureBranch # Pushes a new featureBranch and tracks changesgit pull origin # Tries to update the current branch (merge by default)

Rebase vs Merge

Merge

Create a new commit from two parents and merge the files

Very useful when adding a new feature

Clear point at which it was added (github Pull Requests)

Noisy when overused 

git merge bug-fix # Merges branch 'bug-fix' into the current branchgit merge bug-fix --no-ff # No fast forward merges to show when it was mergedgit merge bug-fix --ff-only # Only merge if fast forward will work

Rebase

“Re-write history"

Squash serval WIP commits into a single commit

Makes you look like you knew what you were doing all along!

Tidy up your commit logs!

  • fixup, squash, reword, edit, and reorder! 
git rebase master # rebase current branch onto mastergit rebase -i HEAD~2 # rewrite the past 2 commitsgit rebase -i master bug-fix # checkout bug-fix branch and rebase onto mastergit pull --rebase # Like regular git pull but tries to rebase (no silly merge)git rebase --abort # If anything goes wrong!

Rebase

Never do it on published (shared) branches

Other Useful Commands

Checkout

Not just for moving between branches!

Grab a particular file from a commit 

git checkout master -- file.txt # Get contents of file.txt from mastergit checkout -- . # discard all unstaged changes

Cherry-pick

Copy a commit to your branch 

git cherry-pick d310dd # Apply the commit with hash d310dd to your branch

Stash

The 'scratchpad' 

git stash # Saves changes to the stashgit stash pop # Applies the top most changesgit stash save "Not Ready" # Gives the stash a better name

Magical -p operator

View changes and split up large edits 

git add -p # Choose what parts (hunks) are addedgit log -p # Show the log with the diffsgit stash -p # Like add for the stashgit show defab4 -p # Show the changes from commit defab4

Reset Soft

Sets the branch to a commit and all differences go to the index

Great way to 'undo' an unpublished commit 

git reset --soft HEAD~1 # Undo the previous commit and keep changes

Reset Hard

Sets the branch to a commit without preserving keeping the index 

git reset --hard HEAD~1 # Undo the previous commit and discard changesgit reset --hard def8ad # Reset the current branch to a commit

Reflog 

git reflog # Can reset to any of the hashes
